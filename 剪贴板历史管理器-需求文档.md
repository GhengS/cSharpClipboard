# 剪贴板历史管理器 — 需求文档（AI / 开发提示词）

本文档描述 Windows 桌面端「剪贴板历史」工具的产品需求与技术约束，可作为 AI 辅助编程提示词或正式需求基线。

---

## 1. 项目概述

开发一个 **Windows 桌面应用**，用于**自动记录并管理剪贴板历史**，支持检索、编辑、删除与**追加粘贴**；强调 **交互流畅、列表滚动顺滑、键盘快捷键跟手**。

---

## 2. 技术栈

| 项 | 要求 |
|----|------|
| 语言 | C# |
| 运行时 | .NET 8（或与 Visual Studio 2022 兼容的当前 LTS） |
| UI | WPF |
| 目标系统 | Windows 10 / 11（x64） |
| 持久化 | 本地存储；**推荐 SQLite**，需考虑大量历史下的查询与索引（必要时 FTS） |

---

## 3. 功能需求

### 3.1 剪贴板采集

- 自动监听系统剪贴板变化并写入历史。
- **首版范围建议**：**纯文本**（`Text` / `UnicodeText`）；若扩展图片/文件，需单独定义大小上限与存储策略。
- **去重**：连续完全相同的内容可不重复入库（可配置）。
- **数量与体积**：可配置「最大保留条数」「单条最大长度」，超限策略需定义（丢弃或截断并标记）。

### 3.2 展示

- 主界面支持 **列表视图** 与 **卡片视图** 两种模式，可切换。
- 用户偏好（当前模式、窗口尺寸等）宜持久化。

### 3.3 搜索

- 支持按内容关键词搜索历史；输入响应快，**不阻塞 UI**（防抖 + 后台查询或索引）。
- 搜索结果高亮或至少清晰可辨（实现阶段再定视觉细节）。

### 3.4 单条操作

- **删除**：单条删除；可选批量删除（若实现需防误触确认）。
- **编辑**：编辑已保存条目内容，保存后刷新列表与持久化数据。

### 3.5 选择与复制

- 选中条目可 **写回系统剪贴板**。
- 可选：复制后自动关闭窗口（设置项）。

### 3.6 追加粘贴

- **定义（需在实现中明确并文档化）**：例如  
  - **方案 A**：将多条选中内容按顺序用换行符（或可配置分隔符）**合并为一个字符串**后放入剪贴板，用户按一次 `Ctrl+V` 粘贴；或  
  - **方案 B**：按顺序多次向目标窗口注入粘贴（依赖 `SendInput` 等，需说明焦点、Unicode、延迟与限制）。  
- 首版推荐 **方案 A**（行为可预期、依赖少）；若做方案 B，须在文档中写明 **Windows 与安全软件限制**。

### 3.7 系统集成（可选但建议）

- 系统托盘：最小化到托盘、托盘菜单退出。
- 全局热键：呼出/隐藏主窗口、必要时上下选择条目（具体键位可配置）。
- 开机自启（可选）。

## 4. 非功能需求（体验与性能）

### 4.1 流畅度

- 历史数百～上千条时，列表/卡片 **滚动须流畅**：必须使用 **UI 虚拟化**（如 `VirtualizingStackPanel`、为 `ItemsControl`/`ListBox` 启用虚拟化等），禁止为每条记录无节制构建完整可视树。
- 启动与打开窗口应尽可能快；避免启动时同步加载全量大文本到内存（可按需懒加载正文）。

### 4.2 快捷键「跟手」

- 全局热键注册与回调须 **轻量**：回调中只做入队/Post 到 UI，**禁止**在热键线程做 IO、大字符串处理或数据库同步操作。
- 窗口内快捷键（方向键、Enter、Esc 等）须 **无可见卡顿**；业务逻辑与磁盘写入异步化。

### 4.3 剪贴板监听正确性

- 使用 **剪贴板变更通知**（如 `AddClipboardFormatListener`），避免纯高频轮询。
- 防止 **自触发循环**：应用自身写入剪贴板时，监听逻辑应能区分或短暂抑制记录（实现细节需在代码注释中说明）。

### 4.4 稳健性

- 处理剪贴板被其他程序占用、打开失败等异常，不崩溃。
- 退出时保证队列内待写入数据落库或安全放弃策略明确。

## 5. 架构与代码组织建议

- **MVVM**：View / ViewModel / Services 分层清晰。
- **服务拆分建议**：`ClipboardMonitorService`、`HotkeyService`、`HistoryRepository`、`SettingsService` 等。
- **依赖注入**：可选；若使用请在 README 中说明注册关系。
- **主线程约定**：WPF Dispatcher 仅用于 UI 更新；数据库与剪贴板读写在后台线程或 async，避免阻塞。
